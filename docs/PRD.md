# Notion 견적서 웹 뷰어 MVP PRD

## 핵심 정보

**목적**: 노션으로 관리하는 견적서를 클라이언트가 전문적인 웹 페이지에서 확인하고 PDF로 다운로드할 수 있게 하는 문제를 해결한다
**사용자**: 견적서를 발행하는 프리랜서/소규모 사업자(관리자)와 견적서를 수신하는 클라이언트(열람자)

---

## 사용자 여정

```
1. [관리자] 노션 데이터베이스에 견적서 작성
   ↓ 노션에서 직접 입력 (서비스 외부)

2. [관리자] 대시보드 페이지 접속 (URL 토큰으로 접근)
   ↓ 견적서 목록 확인

   [견적서 행 클릭] → 견적 항목 테이블 확장 (항목명, 수량, 단가, 소계 확인)
   [같은 행 재클릭] → 축소
   ↓

3. [관리자] 견적서 선택 → 공유 링크 복사
   ↓ 클라이언트에게 링크 전달 (이메일, 메신저 등)

4. [클라이언트] 공유 링크 클릭 → 견적서 열람 페이지
   ↓ 견적서 내용 확인

5. [클라이언트] PDF 다운로드 버튼 클릭
   ↓ PDF 파일 저장

6. [완료] → 견적서 확인 완료
```

## 접근 제어 설계

### 대시보드 보호 방식 (F010)

로그인 UI와 세션 관리 없이, 환경변수 기반 URL 토큰으로 대시보드를 보호한다.

```
접근 흐름:
관리자 -> /dashboard?token={ADMIN_TOKEN} -> Next.js middleware에서 토큰 검증
  ├── 일치 -> 대시보드 렌더링
  └── 불일치 또는 토큰 없음 -> 404 페이지 반환 (존재 자체를 숨김)
```

**환경변수:**
- `ADMIN_TOKEN`: 대시보드 접근용 비밀 토큰 (충분히 긴 랜덤 문자열 권장, 32자 이상)

**구현 위치:**
- `middleware.ts`에서 `/dashboard` 경로 요청 시 쿼리 파라미터 `token` 검증

**보안 고려사항:**
- HTTPS 필수 (Vercel 기본 제공)
- 토큰이 브라우저 히스토리에 남으므로 관리자 개인 기기에서만 사용 권장
- 견적서 열람 페이지(`/quotes/{id}`)는 Notion 페이지 UUID를 사용하므로 추측 불가능
